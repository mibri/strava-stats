# 🏃 strava-stats

A personal running analytics app + LLM coach, built on a Strava data export.
Three parts:

1. **Pipeline** (`pipeline/`) — cleans the raw export, keeps runs only, derives
   metrics, and writes tidy JSON the dashboard and coach read.
2. **Dashboard** (`web/`) — a local web app for progression charts, a run
   explorer, route maps, and fun stats.
3. **Coach** (`coach/` + `/coach`) — an LLM running coach that reads your full
   history. Runs through Claude Code on your subscription (no API key needed).

## Quick start

```bash
# 1. one-time setup
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. build the cleaned data (re-run whenever data/ changes)
python -m pipeline.build

# 3. open the dashboard
python web/serve.py        # → http://localhost:8000/web/index.html
```

## The coach

In Claude Code, run:

```
/coach how is my marathon build tracking?
/coach should I be worried about my IT band given last week's jump?
/coach                     # no question → state-of-training readout
```

The coach reads:
- `coach/goal.md` — your goal & constraints (**edit this by hand** when they change)
- `coach/coach_context.md` — auto-generated training brief (rebuilt by the pipeline)
- `coach/conversations/` — a log it appends to, so it remembers past advice
- the full `data/clean/` dataset for deeper questions

## Data layout

```
data/
  activities.csv            raw Strava export (summary table)
  activities/               raw per-activity tracks (.fit/.fit.gz/.gpx)
  clean/                    ← generated by the pipeline
    runs.json / runs.parquet  one row per run (imperial units, derived metrics)
    streams/<id>.json         per-run splits, best efforts, pace/HR/elev/route
    routes.geojson            all GPS routes (for the map)
    summary.json              aggregates: weekly/monthly, fitness curve, PRs
coach/
  goal.md                   hand-edited goal & profile
  coach_context.md          generated training brief
  conversations/            coach memory log
```

Everything under `data/` is gitignored (it's your private export).

## Excluding bad runs from progression

Some runs have bad GPS/elevation/HR data that skews the trend charts (e.g. a run
with a corrupt grade-adjusted pace). To drop a run from the **progression charts
only** — while keeping it in totals, weekly volume, the calendar, and the map —
add its Activity ID to `progression_excludes.json`:

```json
{ "ids": ["18072746361", "17737070212"] }
```

Find the Activity ID in the runs table (excluded runs show a ⊘) or the run-detail
view, then re-run `python -m pipeline.build`.

## Refreshing with new runs

Re-export from Strava and drop the new `activities.csv` + track files into
`data/`, then re-run `python -m pipeline.build`.

**Live sync (stretch goal):** `pipeline/strava_api.py` is a documented stub for
pulling new activities straight from the Strava API — wire it up when you want
day-to-day freshness without a manual export.
